应用程序缓存 Application Cache

通过创建 cache manifest文件📃

优点：

  • 可离线浏览
  • 已缓存资源加载更快
  • 减少服务器负载：浏览器只从服务器下载更新过的资源

manifest文件

需要设置正确的MIME-type：text/cache-manifest，必须在web服务器上进行配置

三个部分

  • CACHE MANIFEST：在此标题下列出的文件将在首次下载后进行缓存
  • NETWORK：在此标题下列出的文件需要与服务器的链接，不会被缓存
  • FALLBACK：在此标题下列出的文件规定当前界面无法访问时的回退界面（比如404界面）

更新缓存

  • 用户清空浏览器缓存
  • manifest文件被修改
  • 有程序来更新应用缓存
